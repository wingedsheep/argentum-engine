Getting Started

Prerequisites, environment setup, build commands, and the development workflow.

Prerequisites

ToolVersionNotes
JDK21+Kotlin compilation & server runtime
Node.js18+React web client
justLatestRecommended command runner (github.com/casey/just)
DockerLatestOptional — Redis-backed session persistence

Initial Setup

# Clone the repository
git clone <repo-url>
cd argentum-engine

# Build the full project
just build

# Install web client dependencies
just client-install

Environment Configuration

Copy .env.example to .env and configure:

CACHE_REDIS_ENABLED=false           # Enable Redis for session persistence
REDIS_HOST=localhost
REDIS_PORT=6379
GAME_AI_ENABLED=false               # Enable AI opponent (requires OpenRouter)
OPENROUTER_API_KEY=                  # API key for AI opponent
GAME_DEV_ENDPOINTS_ENABLED=true     # Required for E2E tests
GAME_SETS_ONSLAUGHT_ENABLED=true    # Per-set enable flags

Build Commands

Using just (recommended)

CommandDescription
just buildBuild entire project
just testRun all tests
just test-rulesRun rules-engine tests only
just test-serverRun game-server tests only
just test-class ClassNameRun a specific test class
just serverStart game server on localhost:8080
just server-onsStart server with Onslaught enabled
just clientStart React dev server on localhost:5173
just client-installInstall web client npm dependencies
just client-buildProduction build of web client
just client-typecheckRun TypeScript type checking
just cleanClean all build artifacts

Using Gradle directly

./gradlew build                                           # Full build
./gradlew test                                            # All tests
./gradlew :rules-engine:test                              # Engine tests only
./gradlew :rules-engine:test --tests "CreatureStatsTest"   # Specific test class
./gradlew :game-server:bootRun                            # Start server

Web client directly

cd web-client && npm install       # Install dependencies
cd web-client && npm run dev       # Dev server at localhost:5173
cd web-client && npm run build     # Production build
cd web-client && npm run typecheck # Type checking

Running the Application

  1. Start the server: just server
  2. Start the client (separate terminal): just client
  3. Open browser: Navigate to http://localhost:5173

Running E2E Tests

Prerequisite: Both the server (just server) and client (just client) must be running.

Project Directory Structure

argentum-engine/
├── mtg-sdk/                 # Shared contract: DSLs, data models, primitives
├── mtg-sets/                # Card definitions (Portal, Onslaught, Legions, ...)
├── rules-engine/            # Core MTG rules engine (zero server dependencies)
├── game-server/             # Spring Boot: WebSocket, state masking, lobbies
├── web-client/              # React app: browser UI (dumb terminal)
├── e2e-scenarios/           # Playwright E2E tests
├── docs/                    # Architecture documentation (12 files)
├── backlog/                 # Feature backlogs and card tracking
├── assets/                  # UI assets (backgrounds, icons)
├── justfile                 # Task runner commands
├── docker-compose.yml       # Production Docker setup
└── docker-compose.local.yml # Local dev Docker setup

Development Workflow

Adding a New Card

  1. Create card file in mtg-sets/src/.../definitions/{set}/cards/
  2. Define card using the cardDef { } DSL
  3. Add to the set definition file
  4. Write scenario tests for interesting mechanics
  5. Engine loads cards automatically via ServiceLoader

See the Card Development Guide for detailed instructions.

Adding a New Effect

  1. SDK: Add effect data class to mtg-sdk/.../scripting/effects/
  2. SDK: Add factory method to Effects.kt
  3. Engine: Create executor in rules-engine/.../handlers/effects/
  4. Engine: Register executor in the appropriate ExecutorModule
