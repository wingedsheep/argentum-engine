Scenario Tests (Kotest)

Scenario tests use ScenarioTestBase with a builder pattern to set up game state and execute actions:

class LightningBoltTest : ScenarioTestBase() {
    init {
        "Lightning Bolt deals 3 damage to player" {
            val state = setupGame {
                player1 {
                    hand("Lightning Bolt")
                    battlefield("Mountain")
                    library("Mountain")
                }
                player2 {
                    library("Mountain")
                }
            }
            val result = castSpellTargetingPlayer(
                state, "Lightning Bolt", player2Id
            )
            result.assertLifeTotal(player2Id, 17)
        }
    }
}

ScenarioTestBase Helper Methods

MethodPurpose
castSpell(state, cardName)Cast a spell with no targets
castSpellTargetingPlayer(state, cardName, playerId)Cast targeting a player
castSpellTargetingCreature(state, cardName, creatureName)Cast targeting a creature
castSpellTargetingGraveyardCard(state, cardName, targetName)Cast targeting card in graveyard
activateAbility(state, cardName)Activate an ability
declareAttackers(state, creatureNames)Declare attackers
declareBlockers(state, blockerMap)Declare blockers
resolveStack(state)Resolve top of stack
Important: ScenarioTestBase.cardRegistry only includes cards from registered sets (Portal, Onslaught, etc.). Test creatures like “Centaur Courser” from TestCards.kt are NOT available. Instead, create inline test cards with CardDefinition.creature() and register via cardRegistry.register(card).
Always give test players at least one card in library to prevent draw-from-empty losses.

E2E Tests (Playwright)

Full-stack browser tests running against server + client. Located in e2e-scenarios/.

Prerequisites: Server running (just server), client running (just client), and GAME_DEV_ENDPOINTS_ENABLED=true.

Test Structure

import { test, expect } from '../../fixtures/scenarioFixture'

test('Lightning Bolt deals 3 damage', async ({ createGame }) => {
  const { player1, player2 } = await createGame({
    player1Name: 'Alice',
    player2Name: 'Bob',
    player1: {
      hand: ['Lightning Bolt'],
      battlefield: [{ name: 'Mountain', tapped: false }],
      library: ['Mountain'],
    },
    player2: {
      battlefield: [{ name: 'Glory Seeker' }],
      library: ['Mountain'],
    },
    phase: 'PRECOMBAT_MAIN',
    activePlayer: 1,
  })

  const p1 = player1.gamePage
  await p1.clickCard('Lightning Bolt')
  await p1.selectAction('Cast')
  await p1.selectTarget('Glory Seeker')
  await p1.confirmTargets()
  // Auto-resolves if opponent has no responses
  await p1.expectNotOnBattlefield('Glory Seeker')
})

Important Patterns

Auto-Pass

When a player has no legal responses, they auto-pass. No explicit p2.pass() needed unless P2 actually has instant-speed responses.

Stack Items Always Stop Opponent

Opponent always gets priority to see and respond to stack items. Use p2.pass() or p2.resolveStack('Card Name').

Face-Down Creatures

Alt text is "Card back", not the card name. Use clickCard('Card back'). Turn face-up button label is "Turn Face-Up" (hyphenated).

Activated Ability Buttons

Show full ability description text, NOT “Activate”. Use partial text match: selectAction('damage to target').

Aura & Sacrifice Targeting

Uses the ChooseTargets modal. Need confirmTargets() after selectTarget().

MayEffect + Target Triggered Abilities

Engine asks yes/no FIRST, then targets. Test flow: resolveStack() → answerYes() → selectTargets() → resolveStack().
