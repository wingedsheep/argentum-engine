ECS Components Reference

All component types in the Entity-Component-System model. Components are pure data bags attached to entities via ComponentContainer. Tag components (data objects) represent binary state.

ComponentContainer API

container.get<CardComponent>()        // Get or null
container.require<CardComponent>()    // Get or throw
container.has<TappedComponent>()       // Boolean check
container.with(component)              // Add/replace → new container
container.without<TappedComponent>()   // Remove → new container

Identity Components

Located in state/components/identity/. Define what an entity is.

ComponentTypeDescription
CardComponentDataCard name, mana cost, type line, base stats, keywords, colors, oracle text, image URI
OwnerComponentDataCard owner (never changes)
ControllerComponentDataCurrent controller (can change from control effects)
TokenComponentTagEntity is a token, not a card
CopyOfComponentDataTracks what this card is copying
CantBeCounteredComponentTagSpell can’t be countered
FaceDownComponentTagCard is face-down (2/2 creature)
ProtectionComponentDataProtection from colors/subtypes
HexproofFromColorComponentDataHexproof from specific colors
TextReplacementComponentDataText changed (Layer 3)

Components that grant effects to a permanent’s controller or other entities.

PlayWithoutPayingCostComponentMay play without paying mana cost
MayPlayFromExileComponentMay play from exile

Zone Change Behavior

When a permanent leaves the battlefield, stripBattlefieldComponents() removes all battlefield-only components (TappedComponent, DamageComponent, CountersComponent, AttackingComponent, etc.), resetting the entity to its base identity.
