SDK Module

The shared contract layer. Pure serializable data models, DSL builders, effects, triggers, filters, and dynamic values. Zero execution logic — zero dependencies on other modules.

Design Principle: Everything in mtg-sdk is a data structure annotated with @Serializable. The SDK defines what happens; the rules-engine defines how.

Package Structure

com.wingedsheep.sdk/
├── core/           # Enums: Zone, Phase, Step, Keyword, CardType, Color, Subtype
├── model/          # CardDefinition, CardScript, EntityId, CreatureStats
├── scripting/
│   ├── effects/    # Effect sealed hierarchy (damage, life, library, etc.)
│   ├── costs/      # PayCost sealed hierarchy
│   ├── conditions/ # Condition sealed hierarchy
│   ├── values/     # DynamicAmount AST
│   ├── targets/    # EffectTarget, TargetRequirement
│   └── filters/    # GameObjectFilter, TargetFilter, GroupFilter
└── dsl/            # Facades: Effects, Triggers, Conditions, Costs, Filters, Targets
                      # Builders: CardBuilder, SpellBuilder, AbilityBuilders
                      # Patterns: EffectPatterns, LibraryPatterns, HandPatterns

CardDefinition

The master object defining a Magic card. Separates static attributes from behavioral logic.

PropertyTypeDescription
nameStringCard name
manaCostManaCostParsed from strings like "{2}{U}{B}"
typeLineTypeLineCard type, supertype, subtype
creatureStatsCreatureStats?Power/toughness
keywordsSet<Keyword>Flying, Haste, Trample, etc.
scriptCardScriptAll behavioral logic
backFaceCardDefinition?For double-faced cards
startingLoyaltyInt?For planeswalkers
metadataScryfallMetadataRarity, artist, flavor, art URL

Factory methods: creature(), sorcery(), instant(), basicLand(), enchantment(), aura(), artifact(), equipment(), artifactCreature(), planeswalker(), kindredInstant(), kindredEnchantment()

Effects Facade

The Effects object provides factory methods for all effect types. Always use these instead of raw constructors.

// Damage
Effects.DealDamage(3, target)
Effects.Fight(target1, target2)

// Life
Effects.GainLife(4)
Effects.LoseLife(2, EffectTarget.PlayerRef(Player.TargetOpponent))

// Drawing
Effects.DrawCards(2)
Effects.DrawUpTo(3)

// Removal
Effects.Destroy(target)
Effects.Exile(target)
Effects.Sacrifice(target)
Effects.ReturnToHand(target)

// Permanents
Effects.ModifyStats(3, 3, target)
Effects.Tap(target)
Effects.GrantKeyword(Keyword.FLYING, target)
Effects.AddCounters("+1/+1", 2, target)

// Tokens
Effects.CreateToken(Token.Soldier_1_1_White)
Effects.CreateTreasureTokens(3)

// Composites
Effects.Branch(condition, ifTrue, ifFalse)
Effects.MayPay(cost, effect)

See Effects Reference for the complete catalog.

Trigger System

Triggers are built from GameEvent + TriggerBinding (SELF, ANY, OTHER):

TriggerFires When
Triggers.EntersBattlefieldThis permanent enters
Triggers.DiesThis creature dies
Triggers.AttacksThis creature attacks
Triggers.AnyCreatureDiesAny creature dies
Triggers.OnUpkeepBeginning of upkeep
Triggers.OnEndStepBeginning of end step
Triggers.YouCastNoncreatureYou cast a noncreature spell (prowess)

Target System

EffectTarget (resolution-time)

Specifies what an effect acts upon, resolved when the effect executes:

TargetResolves To
EffectTarget.SelfThis permanent
EffectTarget.ControllerThe ability’s controller
EffectTarget.ContextTarget(0)Cast-time target at index 0
EffectTarget.TriggeringEntityObject that caused the trigger
EffectTarget.PlayerRef(Player.Opponent)The opponent
EffectTarget.StoredEntityTarget("exiled")Named variable from earlier effect

TargetRequirement (cast-time)

Declares what a player must select when casting:

// Common targets via Targets facade
Targets.Creature            // Target creature on battlefield
Targets.Player              // Target player
Targets.Any                 // Target any creature or player
Targets.CreatureYouControl  // Target creature you control
Targets.SpellOnStack        // Target spell on the stack

Filter System

GameObjectFilter combines card, state, and controller predicates into a single composable filter:

// Base presets
GameObjectFilter.Creature
GameObjectFilter.Permanent
GameObjectFilter.InstantOrSorcery

// Fluent modifiers
GameObjectFilter.Creature
    .withColor(Color.BLACK)
    .withSubtype("Zombie")
    .withKeyword(Keyword.FLYING)
    .tapped()
    .youControl()
    .powerAtMost(3)

Ability Types

TriggeredAbility

Fires when game events occur. Key properties: trigger, effect, targetRequirements, optional (may), triggerCondition.

ActivatedAbility

Activated by paying costs. Key properties: cost, effect, targetRequirements, isManaAbility, timing.

StaticAbility

Continuous effects applied via Rule 613 layers. Key properties: layer, ability object, affectsFilter, duration.

ReplacementEffect

Intercepts game events before they happen (no stack). Key properties: event (what to intercept), replacement (what happens instead).
