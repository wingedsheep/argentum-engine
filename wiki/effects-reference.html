Effects Reference

Complete catalog of all effect types available in the SDK. All effects are sealed interface subtypes of Effect and are purely serializable data.

Always use the Effects facade (e.g., Effects.DealDamage()) instead of raw constructors (e.g., DealDamageEffect()) in card definitions. Use EffectPatterns for multi-step pipelines.

Library Pipeline (Atomic Effects)

Preferred for library/zone manipulation. Compose from Gather → Select → Move primitives.
EffectDescription
GatherCardsEffectCollect cards into a named collection (no zone change)
SelectFromCollectionEffectPresent choice to split a collection
MoveCollectionEffectMove named collection to a destination zone
RevealUntilEffectReveal from library until filter matches
ShuffleLibraryEffectShuffle target’s library

Pre-Built Patterns (EffectPatterns)

PatternComposed From
Scry(n)Gather(top N) → Select(up to N for bottom) → Move(bottom) → Move(top)
Surveil(n)Same as scry, selected go to graveyard
Mill(n)Gather(top N) → Move(graveyard)
SearchLibrary(filter)Gather(library, filter) → Select → Move(hand) → Shuffle
lookAtTopAndKeep(7, 2)Gather(top 7) → Select(exactly 2) → Move(hand) → Move(rest to GY)
wheelEffect(Each)ForEachPlayer: shuffle hand into library, draw that many
discardCards(n)Gather(hand) → Select(exactly N) → Move(graveyard)
loot(draw, discard)DrawCards → discardCards
destroyGroup(filter)ForEachInGroup(filter, Destroy)
exileUntilEndStep(target)Exile + delayed trigger at end step to return
