Argentum Engine Architecture

Magic: The Gathering Rules Engine • Pure Functional ECS • Data-Driven Design

Pure Functional Core

  • • Immutable GameState — every action returns a new state
  • • ECS architecture — entities + components, no inheritance
  • • Continuation-based async — no blocking threads
  • • Strategy-based registries — pluggable handlers

Data-Driven Cards

  • • CardDefinition = pure data (no execute methods)
  • • Effect AST — serializable, inspectable, no lambdas
  • • Atomic pipelines: Gather → Select → Move
  • • Rule 613 layers for continuous effect projection

  • • Spring Boot 4.0 (Game Server + Gym Server)
  • • React 19 • TypeScript 5.9 • Vite 8
  • • Kotest 6.1 + Playwright (E2E testing)
